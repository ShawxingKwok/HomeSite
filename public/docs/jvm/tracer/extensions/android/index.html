    Most could be static #

    Provide static application context first.

    class App : Application() {
    companion object{
        @SuppressLint("StaticFieldLeak")
        lateinit var context: Context
            private set
    }

    override fun onCreate() {
        super.onCreate()
        context = applicationContext
    }
}

    As App is always initialized at first and destroyed at last, applicationContext could be safely static for common Android developers, which could make most traditionally injected objects also static, likeDatabase, DAO and many related utils.


    Skip if you are using Compose. Learn the common tracer part first if you are not familiar.
